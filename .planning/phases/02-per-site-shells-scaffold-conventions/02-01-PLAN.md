<!-- Empirically verified against a real BASE_PATH build of vfamigos (2026-07-07). -->
<!-- Do NOT re-derive; these facts drive the grep design. -->

SvelteKit (adapter-static, paths.base from BASE_PATH, paths.relative default true) emits:
- RELATIVE asset/link paths in built HTML: `href="./_app/..."`, `href="../"`, `href="./favicon.svg"`.
- Exactly ONE literal `/raj/vfamigos` occurrence per page: the hydration bootstrap JSON `assets: "/raj/vfamigos"` (base-DERIVED, correct — becomes `assets: ""` at custom-domain cutover). This is NOT a leak and MUST NOT trip the guard.

Therefore the guard greps ATTRIBUTE VALUES only, not raw `/raj/`:
  grep -rnE '(href|src)="/[^/]' build --include="*.html"
  - MATCHES (fail): href="/products/", src="/raj/vfamigos/a.png"  (root-absolute, incl. hardcoded /raj/)
  - IGNORES (pass): href="./ok/" (relative), href="//cdn" (protocol-relative), href="https://fonts..." (absolute URL), and the hydration `assets:` JSON (no href=/src= prefix)

Do NOT add a source-level `grep "/raj/"` guard: the Phase-1 placeholder `+page.svelte` contains `/raj/` in a code comment, and a Wave-1-only push (before 02-02/03 rewrite those files) would false-positive and break the build. INFRA-06 governs BUILT HTML — the attribute grep satisfies it precisely.
</interfaces>

    ```yaml
      - name: Portability guard — built HTML             # D-08 / INFRA-06
        working-directory: ${{ matrix.site }}
        run: |
          set -euo pipefail
          # Fail on root-absolute href/src (incl. any hardcoded /raj/... link or asset).
          # Allows ./ relative, // protocol-relative, https:// absolute, and the
          # base-derived hydration `assets:` JSON (not an href/src attribute).
          if grep -rnE '(href|src)="/[^/]' build --include="*.html"; then
            echo "::error::${{ matrix.site }}: root-absolute href/src (incl. hardcoded /raj/) in built HTML — use {base} from \$app/paths or relative paths"
            exit 1
          fi
          echo "portability guard passed for ${{ matrix.site }}"
    ```
